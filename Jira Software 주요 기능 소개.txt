Jira Software 주요 기능 소개

개발 완성도가 떨어지는 소프트웨어들의 결함을 찾기 위함

주요 화면 소개 01 – Jira 초기 화면 소개
결함 추적시스템(BTS)
 - 실시간 관리성(추적성) : 테스터가 결함을 찾으면 개발자들이 바로 해결할수있음(이전에는 동시 진행 불가), 업무의 효율 증가
• Dashboards: 필터를 통해 업무 과제리스트에 대한 현황 표, 차트등을 표시
• Projects: Jira에 등록된 프로젝트를 확인 할 수 있음.
• Issues(많이 사용): 사용자가 참여하고 있는 프로젝트내에 등록된 이슈 및 작업 과제등을 확인 할 수 있
으며, 필터기능을 사용할 수 있음.

Jira - 대시보드
 - Jira의 대시보드는 등록된 이슈들을 기준으로 관리 하고자 하는 요소를 특정하여 Visual 하게 보여줍니다.
 - 대부분의 이슈 적용 기준은 저장된 필터와 연동 됩니다.

주요 화면 소개 02 – 프로젝트 화면 소개
(관리자가 구성)이슈 업무에 대한 상세 내용 각 필드값 입력 내용

이슈 업무와 연관된 사용자 담당자 및 보고자등

주요 기능 소개
이슈 및 업무 유형에 따른 작업 관리
 - 디자인, 운영, VoC, 결함등 업무 유형에 따른 작업 관리 용이
필터 기능
 - 특정 조건에 해당되는 이슈 및 업무리스트를 JQL(Jira Query Language)를 통해 Sorting 다양한 대시보드의 Gadget을 표현 하는데 도움
테스트 관리 기능(AIO Tests or Other TM Plugins)
 - 테스트 케이스를 작성하여 저장 및 테스트 수행 활동에 용이하며, 테스트 단계별 결과를 리포팅하여
 - 버전별 테스트 수행 현황을 파악하는데 용이
Agile 기반 개발 환경 지원
 - Scrum : 개발 수행을 Sprint를 통해 점진적이고 반복적인 개발 수행을 할 수 있게 설정
 - Kanban : 운영 업무에 적합한 과제, 업무를 단계적으로 처리할 수 있는 보드 제공
Workflow
 - 사내 업무 프로세스에 맞는 Workflow를 쉽게 변경 적용 하여, 실무 환경에 맞게 적용

이슈 유형 관리
Jira의 가장 중심인 필드이며, 모든 이슈는 이슈 유형(Issue Type)에 따라 연동 되어 관리 됨.
▪ 이슈 유형(이슈타입)에 따라 업무흐름(워크플로우), 필드 설정, 화면 등을 다르게 설계하여 정책화 함.
▪ 예, 
Test Case 유형에 노출 되는 화면 속의 필드와 결함 유형에 대한 화면 속 필드는 서로 관리 관점이 다르며, 이
를 적절하게 관리 하고 추적 하기 위해 서로 다른 화면 과 업무 흐름을 적용 해야 함.

업무 흐름 관리- Workflow
 - 이슈 유형에 따라 다르게 업무 흐름을 만들 수 있음
 - 이슈를 발견한 테스터가 완료처리하도록 검증 절차를 만들수 있음
 - 다른 사람의 동의 없이 이슈 없애는것 불가
• 프로젝트 별로 업무를 관리하는 흐름을 제어하는 것으로, 해당 이슈의 상태를 변경 하고, 권한 및 담당 관리
를 할 수 있음. 
• 예, 결함에서 완료를 진행 할 경우, “원인＂과＂대책＂을 필수로 지정 해야 한다면, 아래 흐름제어에서 해당
원인과 대책 필드에 적절한 값이 입력 되지 않을 경우에는 완료를 할 수 없게 제어 함.
• Worklog도 강제적으로 입력 할 수 있도록 제어
