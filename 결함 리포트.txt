결함추적활동
등록(이슈) - 검토 - 할당 - 수정 - 종료

결함 수명 주기
(New)결함 등록
(Open)결함 수정 시작
(Fixed)결함 수정 완료
(Retest)결함 수정 확인
 - 수정 여부에 따라 Reopen/Closed 처리
(Closed)결함 수정 확인 완료

결함 리포트 구성요소
1. 유형(Type) : 보고 유형
 - 결함(Bug) : 결함 보고
 - 개선사항(Improvement) : 사용성/서비스 개선사항 보고
 - 이슈(Issue) : 문의사항 등
2. 제목(Summary)
 - 결함의 주요 내용을 확인할 수 있도록 발생 위치 및 오류 사항을 명시하며 요약하여 기술
 - 예시) 설정 화면 이동 버튼 선택 시 진입 안 되는 현상
3. 내용(Description)
 - 결함을 재현하고 확인하는데 필요한 정보를 포함하여 상세하게 기술
 - 최대한 상세하게 기술
 - 예시
재현 버전
재현율
재현 시각
테스트 환경
[사전 조건]
[결함 스탭]
[결함 내용]
[기대 결과]
[참고 사항]
 - 위에 잘 안봄, 최대한 자세히
[첨부파일]
 - 상세하게
4. 결함 심각도(이슈의 위험도) : 응용 시스템에 미치는 영향의 척도
 - Critical : 사용자에게 치명적인 영향을 끼치는 문제
 - Major : 사용 빈도가 높은 페이지나 기능의 오류로 인해 다수의 사용자에게 피해를 줄 수 있는 문제
 - Minor : 기능 동작에는 문제가 없으나, 신뢰를 감소시킬 수 있는 불편한 문제
 - Trivial : 사용 빈도가 낮은 페이지나 기능에서의 사소한 문제
 - 마이너가 디폴트, 주로 결제 관련된 부분은 높이기
 - 고객사 입장에서 이 구조에 따라서 별로 필요없다고 생각한다면 트리비얼
5. 구성요소
 - 프로젝트 내 개발 시스템의 모듈
 - 예시) 메인 화면, 전화 기능, 연락처 기능
6. 담당자
 - 결함 해결 담당자 
7. 참조자
 - 보고되는 결함과 관계되어 있는 이해관계자들을 설정하여 결함 공유 
8. 첨부 파일
 - 결함을 해결하는데 도움이 되는 이미지, 동영상, 로그 파일 등을 첨부
9. 결함 유형
 - 결함의 유형 정보
10. 해결책 : 결함 해결 시 처리 상태를 명시하여, 결함의 결과 확인을 용이하게 함
 - Fixed : 결함 수정한 경우
 - Spec : 기획이 업데이트 안된 내용이었음
 - Not a Bug(Defect) : 결함이 아닌 경우
 - Duplicate : 이미 등록된 이슈인 경우
 - Cannot Reproduce : 재현되지 않는 경우
 - Later : 차기 버전 적용 예정인 경우
 - After Release : 출시 후 적용 예정인 경우

이슈작성 : 우선순위(하이,미디엄,로우)가 있을수도 있고 아닐 수도 있음
