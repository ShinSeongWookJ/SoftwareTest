테스트 케이스 설계 준비
SPECIFICATION
테스트 케이스를 작성하기 위해 기본 베이스가 되는 문서가 몇 가지 있다.
문서들을 기반으로 테스트 케이스를 작성하고, 이렇게 작성한 테스트 케이스를 이용하여 진행하는 테스트 수행 방식을 ISTQB에서는 명세 기반 테스트 기법이라 부르고 있다.
여기서 명세 기반이라는 단어는 바로 테스트 케이스 작성에 기반이 되는 문서를 의미한다.
SPECIFICATION → 테스트 베이시스

Good TC & 설계 유의사항
좋은 테스트 케이스
 - 사용자 요구사항이 모두 포함된 테스트 케이스
 - 오류를 찾아낼 확률이 높은 테스트 케이스
 - 내용이 중복되지 않은 테스트 케이스
 - 너무 간단하거나, 복잡하지 않은 테스트 케이스
TC 설계 유의사항
 - 테스터의 스킬, 능력, 시스템에 대한 이해도를 고려하여 상세하게 설계되어야함
 - 테스트 이전에 이루어져야 하며, 이상적으로 시스템 설계시 시작되어야 함
 - 테스트 케이스에 대해 개발 그룹 간에 베이스라인을 확정해야함
 - 명확한 입력 값이 필요한 경우, 입력 값을 명시해야함
