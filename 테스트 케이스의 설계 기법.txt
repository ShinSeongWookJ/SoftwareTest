테스트 케이스의 설계 기법
1. 명세 기반 기법
기초 : 명세 테스트
설명 : 명세 기반 테스트 케이스 도출(코드 내부에 대한 이해는 요구하지 않음)
종류 : 동등 분할 테스트, 경계값 분석 테스트, 조합 테스트, 결정 테이블 테스트, 상태 전이 테스트, 유즈케이스 테스트
2. 구조기반 기법 
기초 : 구조 테스트
설명 : 프로그램의 소스코드 기반 테스트 케이스 도출
종류 : 제어흐름 테스트, 데이터흐름 테스트
3. 경험 기반 기법
기초 : 경험 테스트
설명 : 개발 시스템 및 서비스에 대한 지식이나 경험으로 테스트 케이스 도출(과거 유사 프로젝트에 대한 경험, SW 내 자주 발생하는 결함 분포 등의 지식 토대)
종류 : 탐색적 테스트, 오류 추정, 체크리스트

화이트박스 테스트 & 블랙박스 테스트
1. 블랙박스 테스트
 - 명세 기반 기법
 - 소프트웨어의 내부 구조를 직접 참조하지 않고, 요구사항 명세서나 설계 문서를 기반으로 소프트웨어의 기능적 혹은 비기능적 요구 및 지시사항이 제대로 동작하는지 검증하는 방법
2. 화이트박스 테스트
- 구조 기반 기법
- 컨포넌트 혹은 구조와 같은 내부 구조 분석에 바탕을 두고 검증하는 방법
※ 실무에서는 블랙박스 테스트 중심으로 이루어질 예정
